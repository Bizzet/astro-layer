# seo-optimize

Adds or improves SEO metadata, Open Graph tags, and structured data for Astro pages.

## Per-Page Requirements

Every page must have:
- Unique `<title>` — format: `Page Name | Business Name` (under 60 characters)
- Unique `<meta name="description">` — 150–160 characters, descriptive
- `<link rel="canonical" href={Astro.url} />`
- OG tags: `og:title`, `og:description`, `og:image`, `og:type`

## LocalBusiness JSON-LD (Homepage)

The homepage must include LocalBusiness structured data:

```json
{
  "@context": "https://schema.org",
  "@type": "LocalBusiness",
  "name": "Business Name",
  "address": {
    "@type": "PostalAddress",
    "streetAddress": "123 Main St",
    "addressLocality": "City",
    "addressRegion": "ST",
    "postalCode": "12345"
  },
  "telephone": "+1-555-000-0000",
  "openingHours": "Mo-Fr 08:00-18:00",
  "geo": {
    "@type": "GeoCoordinates",
    "latitude": 0.0,
    "longitude": 0.0
  },
  "url": "https://example.com",
  "image": "https://example.com/og-image.jpg"
}
```

Pass as `jsonLd` prop to the Base layout — the SEO component serializes and injects it.

## Sitemap and robots.txt

- Sitemap auto-generated by `@astrojs/sitemap` at `/sitemap-index.xml`
- Confirm `@astrojs/sitemap` is in `astro.config.mjs` integrations
- `public/robots.txt` — `Allow: /` by default

## Optional: Content Security Policy

Astro 6 ships stable `security.csp` support in `astro.config.mjs`. For high-trust clients, mention this as an available hardening option — but do not auto-generate it as the rules are project-specific.
